基于lvs-dr模型的discuz负载均衡实现

发表于 2016-05-09   |   分类于 Cluster   |  

说明

本文并不涉及lvs工作原理的介绍和论述,而仅是记录基于lvs-dr模型的负载均衡的实现过程.需要了解更多技术细节请参照博文《VS的工作模型和调度算法介绍》

规划

  • lvs-dr模型与之前的lvs-nat模型基本情况差不多,只是网络拓扑结构不同.

四台主机需要初始化成如下状态.它们都是同一台宿主机上的虚拟机,均是最小化安装,iptables和SELinux均处于关闭状态.四台主机均只有一个物理网口,原始IP地址均处于同一个网段172.18.71.0/16.其中HostA在网口别名eth0:0上配置VIP向用户提供服务,HostB和HostC在网口别名lo:0上需要关闭其ARP响应,然后也配置上VIP地址.HostB和HostC作为两台RS提供真实的web服务,HostD作为HostB和HostC的后端共享存储提供mariadb和NFS.

2016-05-16 更新
知道为什么会话可以保持了,是因为在Discuz! X中一如继往的,SESSION并没有使用PHP自带的SESSION机制，而是系统的一套自带的机制。实现细节请参照《Discuz! X 中 SESSION 机制讲解》
