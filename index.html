pyenv--python虚拟开发环境

发表于 2016-08-11   |   分类于 python   |  

主要步骤可参照此文章用pyenv 和 virtualenv 搭建单机多版本python 虚拟开发环境, 补充一些自己填的坑.

pyenv install <version>时要去国外站点去下载python相应版本的源码包, 相当之慢. 可以使用国内七牛云存储自建一个镜像站点The pyenv’s official mirror of Python archives, 会快一些, 不过它仅提供稳定的版本, 太新的版本可能就没有, 不过一般也不会那么激进地上新版本.

而且如果编译失败, 解决完错误, 再次执行pyenv install <version>, 不论你之前是否已经下载过源码包, 它会再次下载. 这个真是坑, 你给个可以指定本地安装包的选项也好啊(也可能有, 但是我没有发现, 知道的同学分享一下.).
这个问题解决起来比较麻烦, 除了换快点的镜像源这种方式外, 可能就只能是自己改代码了. pyenv是用shell写的, 可以改. 不过这个过于复杂, 之前改过一次, 再次碰到时已经完全忘记怎么弄的了.

不过刚刚发现了这么一个工具pyenv_mirror, 现在想想可以这么解决, 就是搭一个本地的镜像站点. 网速实在太慢或者是公司里面需要重度使用pyenv的同学可以尝试自己搭一个, 我这里就不搞了.
