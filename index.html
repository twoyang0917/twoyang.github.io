LAMP--CentOS-6平台双机FastCGI模型

发表于 2016-04-20   |   分类于 Server   |  

需求

解析

  • 由于httpd是cpu密集型应用，php是io密集型应用，而mariadb既是cpu密集型又是io密集型。所以将httpd+php部署在一台主机上，mariadb部署在另一台主机上。
  • 由于CentOS-6平台仅提供mysql的rpm安装包，这里我们使用mariadb的二进制版本安装包。
  • 由于CentOS-6平台提供的各应用的rpm安装包版本都比较老，主要特性都没有提供，所以各应用程序无法直接使用yum源安装，而需要通过编译源码来安装。
  • 因为php要作为独立的服务单独运行，因此编译php时需要启用php-fpm特性。
  • httpd与php之间需要通过FastCGI协议来连接，httpd其实是作为作为反向代理来工作的，编译httpd时需要启用proxy和proxy_fcgi特性。

额外的注意事项：

  • 如要求mpm为event模型，同时php作为httpd的模块工作时。因为event为线程模型，所以php必须启用线程安全功能--enable-maintainer-zts，以下为官方解释。这种情况编译httpd时无需启用代理特性。

LAMP--CentOS-7平台三机FastCGI模型

发表于 2016-04-20   |   分类于 Server   |  

需求

解析

  • 由于CentOS-7平台提供了以上各应用的rpm安装包，且均包含了各应用的新特性，所以各应用程序可以直接使用yum源安装，而无需通过编译源码来安装。
  • xcache为EPEL源中提供的应用程序，因此需要配置启用EPEL源。
  • 因为php要作为独立的服务单独运行于一台服务器上，因此不在是安装php，而是要安装php-fpm了。
  • httpd与php之间需要通过FastCGI协议来连接，httpd其实是作为作为反向代理来工作的，httpd中就需要加载mod_proxy和mod_proxy_fcgi模块了。
