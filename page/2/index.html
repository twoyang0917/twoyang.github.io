keepalived_nginx实现discuz负载均衡和高可用

发表于 2016-05-12   |   分类于 Cluster   |  

说明

此文需要配合下一篇博文《keepalived_tengine实现discuz负载均衡和高可用》一起服用,疗效更好.

此次实验使用nginx替代lvs作为前端调度器,使用keepalived对nginx做双主模型高可用向后做反向代理.中间的RS服务器使用httpd+php来处理php页面请求,httpd和php以模块方式结合.暂不考虑会话保存服务器问题和动静资源分离的问题.

单主模型与双主模型

  • 单主模型(主/备模型),即一台主机为MASTER,另一台主机为BACKUP.作为BACKUP的主机不会工作,只有当MASTER主机宕机,这时BACKUP主机取而代之成为MASTER才会开始工作.因此在单主模型始终有一个主机不工作,会造成资源浪费.
  • 双主模型无非就是两个单主模型叠加,两台主机互为主备.这样再在前端DNS解析时加上双A记录,即可实现两台主机同时工作,既实现了高可用同时还兼具负载均衡.

规划

前端使用两台nginx服务器作为前端调度器,使用keepalived对nginx做双主模型高可用.中间两台httpd+php服务器作为RS服务器,后端一台共享存储提供mariadb和NFS服务.因为不再使用LVS调度,而是通过nginx反向代理来实现,所以中间的RS服务器上不需要在本地回环网口的别名上配置VIP了.
